## 工程方向信号

### Base — 证明系统现代化 + 运维工具链建设

1. **证明系统模块化重构**：proposer 侧提取 proof collector（#3239）和 proof submitter（#3240/#3241），prover-service 迁移至 worker queue 架构（#3193）并清理 outbox 旧代码（#3191），统一重试配置（#3247），challenger 统一 proof retry（#3248）。这是一次系统性的 proof 流水线重构，将分散的证明逻辑整合为清晰的模块边界。

2. **basectl 工具链成熟化**：rpc.rs 拆分（#3209）、公共模块重构（#3216）、新增 block 子命令（#3238）、Kubernetes pod 监控（#3237）。basectl 正在从临时脚本演进为完整的节点运维平台。

3. **性能测试体系**：全新 Rust benchmark crate（#3229 draft）、负载测试真实代币支持（#2732）、zk-benchmark profile types（#3231 draft）。Base 在系统性构建端到端性能度量能力。

4. **Privacy 分支同步**：#3236 的大规模合入表明 privacy 功能开发在持续推进，B20 precompile 合并为 B20Asset，可能面向 Beryl 升级。

### Optimism — Kona 全面接管 + Proof 性能优化

1. **op-program → Kona 迁移**：fault proof 测试迁移至 kona（#21193）、op-e2e 使用 kona-host 替代 op-program（#21217）、移除 Cannon game type（#21254）、清理 op-program prestate 支持（#21214）。这是一次明确的架构过渡，Kona 正在成为 Optimism 的唯一 proof guest。

2. **op-reth proof backfill 加速**：snapshot hashed-leaf 表（#21088）和 snapshot 集成到 backfill（#20828）两个 PR 共同解决 proof backfill 的性能瓶颈，是 op-reth 作为执行客户端的关键优化。

3. **Superroot 和 Interop 演进**：支持 super root withdrawal proving（#21143）、非 interop 链提供 superroot_atTimestamp RPC（#20581）、interop failsafe 可观测性（#21151）和 dedicated RPC 错误码（#21205）。Superroot 架构正在从 interop 专用扩展到通用支持。

4. **CI/基础设施优化**：sccache 编译缓存（#21227）、CI gate 整合（#21237）、命令去重（#21245）。Rust 构建速度是 Optimism 的持续关注点。
